Exam Tips & Strategies

üîê

Master the Shared Responsibility Model

This is tested heavily! Know what AWS manages vs. what you manage. Security OF the cloud (AWS) vs. Security IN the cloud (You).

üí∞

Understand Pricing Models

Know the differences: On-Demand, Reserved (1 or 3 years), Spot Instances, Savings Plans. When to use each one.

üåç

Know the Global Infrastructure

Understand Regions, Availability Zones, Edge Locations. Know which services are global vs. regional.

‚ö†Ô∏è

Read Questions Carefully

Watch for keywords like "most cost-effective," "highly available," "fastest." They indicate the correct answer approach.

üéØ

Use Process of Elimination

Often 2 answers are obviously wrong. Narrow it down to 2 choices, then pick the one that best matches the question's focus.

üö´

Don't Overthink

This is a foundational exam. If an answer seems too complex or requires deep technical knowledge, it's probably not right.

Must-Know Concepts

üîÑ Shared Responsibility Model

AWS Responsibility: Physical security, hardware, networking, virtualization (Security OF the cloud)

Your Responsibility: Data encryption, IAM, OS patches, firewall configuration, application security (Security IN the cloud)

üèóÔ∏è Well-Architected Framework Pillars

1. Operational Excellence
2. Security
3. Reliability
4. Performance Efficiency
5. Cost Optimization
6. Sustainability

üìç Global vs Regional Services

Global: IAM, Route 53, CloudFront, WAF, Organizations
Regional: EC2, RDS, S3 (buckets), Lambda, VPC

üí≥ Support Plans

Basic: Free, billing support
Developer: $29/mo, email support
Business: $100/mo, 24/7 phone
Enterprise: $15K/mo, TAM
