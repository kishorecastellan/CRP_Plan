KMS, Secrets Manager, SSM
KMS key types (symmetric/asymmetric), envelope encryption, SSM Parameter Store (standard vs advanced), Secrets Manager rotation, ACM certificates, encryption in transit vs at rest
Week 7: CodePipeline & CodeBuild

buildspec.yml, appspec.yml, artifacts, CodePipeline stages and actions, CodeBuild environment variables, caching, VPC integration, cross-account deployments
Week 8: CodeDeploy & SAM

Containers, Messaging & Observability

Week 12: Observability

üîç DynamoDB Operations
Query
Uses partition key (required) + optional sort key. Most efficient.
Scan
Reads entire table. Use ProjectionExpression & FilterExpression to limit.
BatchWriteItem
Up to 25 PutItem/DeleteItem, max 16 MB total, max 400 KB/item
BatchGetItem
Up to 100 items, max 16 MB returned
TransactWriteItems
Up to 100 items, ACID transactions, 2x WCU cost

üåê API Gateway Types

WebSocket API
Two-way communication. Routes: $connect, $disconnect, $default

Payload Size
Max 10 MB

SigV4 signed requests. Good for internal services, cross-account.
Lambda Authorizer
TOKEN-based (bearer) or REQUEST-based (headers, query strings). Cache TTL: 0-3600s
Cognito Authorizer
Validates JWT tokens from Cognito User Pool. REST API only.
Stage Variables
Use ${stageVariables.variableName} in Lambda ARN, HTTP endpoint

üå± Elastic Beanstalk Deployments
All at Once
Fastest. Brief downtime. Good for dev.
Rolling
Updates in batches. Reduced capacity during deployment.
Rolling with Additional Batch
Maintains full capacity. Spins up new instances first.
Immutable
New ASG with new instances. Safe rollback. Higher cost.
Blue/Green
Separate environment. Swap URLs. Zero downtime.
.ebextensions
YAML/JSON config files in .ebextensions/ folder. Must end in .config

Pro Tips for Experienced Practitioners

Leveraging your 7+ years of AWS experience for exam success

01
Don't Rely on Experience Alone
The exam tests specific limits, timeouts, and configurations that you may not encounter daily. Memorize Lambda timeout limits, SQS visibility timeouts, DynamoDB capacity calculations. These are frequently tested.
02
Focus on Serverless First
DVA-C02 heavily emphasizes serverless. If your experience is more EC2/ECS-focused, spend extra time on Lambda event source mappings, Step Functions error handling, and API Gateway integration types.
03
Master the CI/CD Tools
Know buildspec.yml and appspec.yml structure cold. Understand the difference between CodePipeline actions, CodeBuild phases, and CodeDeploy hooks. Many questions test "which file/phase/hook" scenarios.
04
Security is 26% of the Exam
Don't underestimate security. Know when to use IAM policies vs resource policies, understand Cognito User Pools vs Identity Pools, and master KMS envelope encryption. STS AssumeRole scenarios are common.
05
Understand "Why" Not Just "How"
Questions often present scenarios where multiple options seem valid. Understanding trade-offs (REST vs HTTP API, SQS Standard vs FIFO, DAX vs ElastiCache) helps eliminate wrong answers.
06
Take Practice Exams Seriously
With 7+ years experience, don't skip practice exams. Use Tutorials Dojo or AWS official practice questions. Aim for 80%+ consistently before scheduling. Review every wrong answer thoroughly.
